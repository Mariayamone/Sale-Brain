# [DEC-003]: Gemini 2.0 Flash for AI Responses

**Date:** 2025-05-27  
**Status:** `Accepted`  
**Author:** Development Team  
**PR Reference:** N/A (Initial project setup)

## Context

The Telegram bot needs AI capabilities to:
1. Understand natural language product queries
2. Generate contextually appropriate responses
3. Assist with cart management conversationally
4. Create marketing campaign content

We needed an AI provider that was:
- Capable of natural language understanding
- Cost-effective for potential high usage
- Easy to integrate
- Fast response times

## Decision

We chose **Google Gemini 2.0 Flash** via the `@google/genai` SDK.

```typescript
import { GoogleGenAI, Type } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });

const response = await ai.models.generateContent({
  model: "gemini-2.0-flash",
  contents: [{ role: "user", parts: [{ text: userMessage }] }],
});
```

### Model Selection Rationale

- **2.0 Flash**: Fast, cost-effective, modern model
- Sufficient for short conversational responses
- Can handle product descriptions and order flow conversations

## Alternatives Considered

- **Option A: OpenAI GPT-4o** — More capable but:
  - Higher cost per token
  - Slower responses
  - Required different API client

- **Option B: Claude (Anthropic)** — Great for longer content but:
  - Higher latency
  - Less integrated with Google's ecosystem
  - Marketing content generation is good but overkill

- **Option C: Rule-based bot (no AI)** — Would be faster but:
  - Less natural conversation
  - Can't handle unexpected user queries
  - High maintenance for conversation flows

## Consequences

- **Positive:**
  - Free tier available (generous limits)
  - Fast inference (Flash model)
  - Good enough for conversational commerce
  - Simple SDK integration

- **Negative:**
  - Not fine-tuned for Burmese/e-commerce
  - May generate inconsistent responses
  - Requires API key management
  - Rate limits apply
