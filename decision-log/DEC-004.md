# [DEC-004]: React 19 with TypeScript

**Date:** 2025-05-28  
**Status:** `Accepted`  
**Author:** Development Team  
**PR Reference:** N/A (Initial project setup)

## Context

We needed a frontend framework that:
- Provides reactive UI updates
- Works with TypeScript for type safety
- Supports modern features
- Has good developer experience

The project is a dashboard for shop management with:
- Multiple pages (onboarding, orders, products, marketing)
- Interactive components with animations
- Form handling for shop configuration

## Decision

We chose **React 19 with TypeScript** using:

- **React 19**: Latest version with improved performance and new hooks
- **TypeScript**: For type safety across components and API
- **Vite**: For fast development and optimized builds

```json
{
  "dependencies": {
    "react": "^19.0.1",
    "react-dom": "^19.0.1"
  },
  "devDependencies": {
    "typescript": "~5.8.2",
    "vite": "^6.2.3"
  }
}
```

### Why React 19 (and not older versions or alternative frameworks)

- React 19 is the latest stable version
- Better performance with automatic batching
- New `use` hook for promise unwrapping
- Improved TypeScript support in JSX transform

## Alternatives Considered

- **Option A: React 18** — More mature but:
  - React 19 is now stable
  - 19 has better performance out of box
  - No significant ecosystem maturity difference

- **Option B: Vue.js** — Good alternative but:
  - Team familiarity with React
  - Larger ecosystem for integrations
  - More component libraries available

- **Option C: SolidJS** — Faster but:
  - Smaller ecosystem
  - Different paradigm to learn
  - Less library support

## Consequences

- **Positive:**
  - Type safety with TypeScript
  - Large ecosystem and community
  - Modern hooks and patterns
  - Good developer experience in VS Code

- **Negative:**
  - Some breaking changes from React 18
  - Some libraries not yet React 19 compatible
  - New patterns to learn (like `use` hook)
