# [DEC-005]: Motion Library for Animations

**Date:** 2025-05-28  
**Status:** `Accepted`  
**Author:** Development Team  
**PR Reference:** N/A (Initial project setup)

## Context

The application requires smooth animations for:
- Onboarding wizard transitions
- Chart animations
- Page transitions
- Interactive UI feedback

We needed an animation library that:
- Works well with React
- Has good performance
- Is easy to use
- Supports modern React features

## Decision

We chose **Motion** (the library formerly known as Framer Motion) via `motion` npm package.

```typescript
import { motion, AnimatePresence } from "motion/react";

<motion.div
  initial={{ opacity: 0 }}
  animate={{ opacity: 1 }}
  transition={{ duration: 0.3 }}
>
  Content
</motion.div>
```

### Why Motion (not other options)

- Formerly Framer Motion - proven and stable
- First-class React support
- Great TypeScript support
- Simple declarative API

## Alternatives Considered

- **Option A: Plain CSS animations** — Would work but:
    - Harder to orchestrate complex sequences
    - Less control over enter/exit states
    - No gesture handling

- **Option B: React Spring** — Good physics-based animations but:
    - Steeper learning curve
    - Less actively maintained
    - Different API paradigm

- **Option C: Native Web Animations API** — Most lightweight but:
    - Verbose syntax
    - Less React-idiomatic
    - More boilerplate code

## Consequences

- **Positive:**
  - Declarative animations
  - Great enter/exit animations
  - Layout animations (AnimatePresence)
  - Gesture support built-in
  - Good documentation

- **Negative:**
  - Adds bundle size (~30KB)
  - Some learning curve
  - Can cause performance issues if overused

## Rule Sync (if applicable)

This decision is reflected in component imports like:

```typescript
// Onboarding.tsx
import { motion, AnimatePresence } from "motion/react";

// TelegramSimulator.tsx  
import { motion } from "motion/react";
```
